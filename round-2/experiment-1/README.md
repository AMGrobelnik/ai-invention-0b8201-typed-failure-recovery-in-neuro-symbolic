## Layman Summary

A neuro-symbolic pipeline that detects why a logic proof failed (wrong word, missing fact, wrong structure), calls an AI to fix exactly that problem, and compares this targeted approach against two standard baselines on real reasoning benchmarks.

## Full Summary

Implements the Typed Unification Failure Recovery Pipeline — a neuro-symbolic reasoning system that classifies symbolic proof failures into four typed categories (lexical mismatch, arity mismatch, missing fact, entity re-identification) and dispatches targeted LLM repairs for each type, compared against two baselines (ARGOS: single generic abductive repair; Logic-LM: raw error forwarding). Evaluated on ProofWriter (propositional logic, true/false queries) and CLUTRR (kinship reasoning) benchmarks.

Key results on 94 full ProofWriter examples + 3 CLUTRR mini examples:
- TYPED accuracy: 89.36% (vs ARGOS 79.79%, Logic-LM 88.30%)
- TYPED outperforms ARGOS by +9.57pp — confirming typed failure dispatch beats undifferentiated abductive repair
- Total LLM cost: $0.0199 (88 calls, model: anthropic/claude-haiku-4-5, budget: $10)
- Hallucination rate: TYPED=76.6%, ARGOS=64.9%, Logic-LM=77.7% (LLM fallback path dominates due to relational ProofWriter sentences parser does not handle)
- SentBERT threshold sensitivity analysis included (thresholds 0.6–0.9)
- Ablation study and sample traces included in method_out.json

Core components: pure-Python forward chainer (KnowledgeBase, Atom, ProofResult), typed failure detector (types 1–4), repair engine (type-specific LLM prompts + generic fallback), SentBERT similarity scorer (all-MiniLM-L6-v2 with bigram fallback), bridge axiom cache (18 entries), CLUTRR kinship graph traversal with BFS and KINSHIP_COMPOSITION table. OpenRouter API with in-memory caching and cost tracking.

Files: method.py (orchestration), src/forward_chainer.py, src/pipeline.py, src/repair_engine.py, src/llm_client.py, src/proofwriter_parser.py, src/clutrr_parser.py, src/sentbert_sim.py, config.yaml, pyproject.toml (all deps pinned to exact versions from .venv).
